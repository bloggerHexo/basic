為何 Xuite 出現了 Facebook 留言框？要如何關閉？

最近 Xuite 在文章下方強制新增了一個 FB 留言框，當然官方立意良善，知道 FB 留言框的社群傳播效益較佳，放在文章下方的留言機會較高。不過是利是弊其實很難講，因為痞客邦(Pixnet)早已這麼做，而根據 WFU 過去的處理經驗，這件事的好處不見得能抵銷掉壞處。

以下先說明這件事的利弊及影響，接著再說明關閉的步驟及建議方案。



一、FB 留言框 的利弊


直接說結論，這個留言框一開始看很新鮮，用一段時間後你一定很生氣，所以也可以直接跳到「二、如何關閉 FB 留言框」。

其缺點有下列這些──

1. 不能管理留言

因為這個 FB 留言框無法判定擁有者是誰，是誰安裝的，所以完全無法給予管理者的權限。


2. 不一定能看到所有人的留言

由於 FB 有隱私權的設定，就算有人留了言，也得剛好是你的朋友(雙方互加好友)，才有權限看到留言內容。如果是路人留言，那麼多半是看不到該則留言，除非剛好那個路人完全沒設定 FB 隱私權，是公開的狀態，那麼才能看到路人的留言。


3. 留言不會通知

因為對這個 FB 留言框不具備擁有者的身份，有人留言你也不會知道，除非部落格每篇文章每天巡一次(誰會這麼做...)，所以要看到這個 FB 留言框有留言，需要抱著隨緣的心態。



二、如何關閉 FB 留言框


因此，對於這個「雞肋」留言框，如果想要關掉的話，請見下圖：



到後台 → 日誌基本設定 → facebook 回應 → 不啟用

然後儲存即可。然而，關閉之後，有更好的選擇方案嗎？



三、FB 留言框 + email 通知


為了解決「無法看到留言」、「無法收到留言通知」這樣的問題，WFU 寫了一個小工具「Xuite 安裝 Facebook 留言板 + 留言 email 通知 + 自適應寬度」，請按這篇文章的操作說明進行即可。



四、各種留言分頁


除了 FB 留言框之外，如果還想安裝多個留言分頁，例如「G+ 留言框」，請參考這篇「幫 Blogger 安裝三種留言分頁 (含 Google+ 及 Facebook 留言板)」的效果，目前所寫的是 Blogger 版本。

因為 WFU 的業務目前主要在 Blogger，還不清楚 Xuite 讀者有無安裝留言分頁的需求，如果需要的話請再告知，會另外釋出 Xuite 的版本。
